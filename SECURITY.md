# Security Policy

## Supported Versions

Only the latest release on the `main` branch receives security fixes during the early development phase.

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead, email the maintainer directly:

**disheng2044@gmail.com**

Include in your report:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (optional)

You will receive an acknowledgment within 72 hours. We aim to release a fix within 14 days for critical issues.
